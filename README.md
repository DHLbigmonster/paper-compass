# Paper Compass

`Paper Compass` 是一个面向实证类学术论文的开源工作流包，可用于 Codex，也可作为 Claude Code 中的项目级提示词与流程规范使用。

它不是代写器。它的核心作用是把一篇需要数据实验的论文拆成清晰、可执行、可检查的阶段，让 AI 帮你做执行、整理、代码生成和交叉检查，而关键学术判断仍由你本人完成。

## 它能帮你做什么

- 帮你判断论文目前处于哪个阶段
- 帮你规划文献阅读路径和阅读笔记结构
- 帮你梳理选题、创新点和研究 gap
- 帮你搭建实验框架与决策点清单
- 帮你设计特征、数据清洗方案和可信度检验
- 帮你生成实验代码、拆分模型训练步骤、选择评价指标
- 帮你组织描述性统计、结果分析、讨论和稳健性检验
- 帮你拆解论文正文写作顺序
- 帮你处理 LaTeX 转换、引用整理和降低 AI 痕迹

## 它不能替你做什么

- 不能替你直接完成一篇你自己不理解的论文
- 不能跳过文献阅读、实验设计和结果核查
- 不能替代导师、审稿人或学术伦理要求
- 不适合纯文科类论文、课程短作业或只需润色的成稿

## 适用场景

- 毕业论文
- 实证论文
- 量化研究
- 需要数据实验的学术写作
- 希望把 Claude Code、Codex、ChatGPT、NotebookLM 组合起来使用的论文工作流

## 使用规则

- 人做决策，AI 做执行
- 首次进入时先判断阶段，不要从头重复已经完成的工作
- 每次只推进当前一步，不要一次性生成整篇论文
- 关键节点必须停下来确认用户是否真正理解
- 重要方案要交叉验证，不依赖单一模型
- 先跑通，再优化
- 如果前置步骤没做好，应回退修正，而不是硬往后推

## 工作流结构

整个流程分为五个阶段：

1. 前期准备
2. 数据实验
3. 结果分析
4. 正式写作
5. 格式调整与降低 AI 痕迹

详细内容在 `references/` 中按阶段拆分，`SKILL.md` 负责定义总流程、判断逻辑、引导方式和检查点。

## 在 Codex 中使用

1. 克隆仓库：

```bash
git clone https://github.com/DHLbigmonster/paper-compass.git
```

2. 将仓库放入你的 Codex skills 目录，例如：

```bash
mkdir -p "$CODEX_HOME/skills"
cp -R paper-compass "$CODEX_HOME/skills/paper-compass"
```

3. 在 Codex 中显式调用：

```text
使用 $paper-compass 帮我从零开始推进一篇实证论文，先判断我当前处于哪个阶段。
```

## 在 Claude Code 中使用

Claude Code 没有必要和本仓库强绑定成 Codex 专属 skill。更合适的方式是把它当成项目内的工作流规范使用。

推荐做法：

1. 克隆仓库到你的论文项目目录，或放在一个你常用的 prompts/workflows 目录中。
2. 在会话开始时要求 Claude Code 先阅读 `SKILL.md`，再按需要读取 `references/` 对应阶段文件。
3. 明确告诉 Claude Code：它的角色是论文教练和执行助手，不是代写器。

推荐起手提示词：

```text
请先阅读 paper-compass 仓库中的 SKILL.md，并把它作为本次协作的工作流规则。
先不要直接写论文，先判断我当前处于哪个阶段。
如果进入某个阶段，再读取 references/ 下对应的阶段文件。
每次只推进一步，给我可以直接执行或复制的下一步动作。
```

## 推荐配合工具

- NotebookLM：文献阅读和问答
- Claude / ChatGPT：方案设计与交叉检查
- Claude Code / Codex / Cursor：代码执行、调试、批处理
- Overleaf：LaTeX 写作与排版

## 输入越完整，效果越好

建议至少提供以下信息中的一部分：

- 论文题目或研究问题
- 已读文献数量与方向
- 数据来源、字段结构、样本量
- 当前已经完成到哪一步
- 学校、导师、期刊或格式要求

## 仓库结构

- `SKILL.md`：核心工作流定义
- `references/`：按阶段拆分的详细说明
- `agents/openai.yaml`：Codex UI 元数据
